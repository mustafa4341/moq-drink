## Adım 1: Merkezi İçecek Veri Dosyası
**Dosya:** `src/lib/drinks.ts` (YENİ)

Tüm 8 içeceğin verilerini tek yerde tanımla:

```typescript
interface MoqDrink {
  id: string;           // "merida", "cool-lime", etc.
  name: string;         // "MERIDA"
  emoji: string;        // "🍊"
  color: string;        // "#E58A2B"
  bgColor: string;      // tailwind class
  textColor: string;    // tailwind class
  glowColor: string;    // rgba for box-shadow
  gradient: string;     // CSS gradient for background
  image: string;        // "/images/merida.png" (placeholder)
  personality: string[]; // 2-3 varyasyon metni
  joke: string;         // Espri
  traits: { cesaret: number; dinginlik: number; nese: number; ferahlık: number }; // 0-100
  ingredients: string[];
  worldName: string;    // "Kaşif", "Sakin", "Lider", etc.
}
```

Her içeceğin 2-3 kişilik metni varyasyonu, espri, enerji profili ve malzemeleri olacak.

## Adım 2: Algoritma Motoru
**Dosya:** `src/lib/mood-algorithm.ts` (YENİ)

### 5 Faktörlü Ağırlıklı Puanlama Sistemi:

| Faktör | Ağırlık | Açıklama |
|--------|---------|----------|
| İsim + Soyisim Hash | %30 | SHA-like hash → hangi içeceğe hangi puan |
| Doğum Mevsimi | %25 | İlkbahar → Merida/Cool Lime, Yaz → Redline/Merida, Sonbahar → Sundrop/Sunset, Kış → Limonata/Sunset |
| Doğum Ayı | %20 | Ayın kendisi ekstra puan dağılımı etkiler |
| Doğum Günü | %15 | 1-10 (sakin), 11-20 (dengeli), 21-31 (enerjik) |
| Gizli MOQ Seed | %10 | `"MOQ_IS_DIFFERENT_2026"` hash → hiç kimsenin çözemeyeceği rastgele katkı |

### Algoritma Adımları:
1. **Hash fonksiyonu:** İsim + soyisim'den deterministik hash oluştur
2. **Mevsim puanı:** Ay → mevsim → o mevsimin içeceklerine puan ekle
3. **Ay puanı:** Doğrudan ay numarasından her içeceğe küçük puan ekle
4. **Gün puanı:** Gün aralığından enerji tipi → bu enerjiye yakın içeceklere bonus
5. **Secret seed:** Gizli string'in hash'i → her içeceğe mikro puan
6. **Topla:** En yüksek puanlı içeceği seç
7. **Varyasyon:** İçeceğin hangi kişilik metnini göstereceğini hash ile belirle (aynı kişi aynı metni görür)

### Easter Egg'ler:
- **Churchill:** Tüm puanlar birbirine çok yakınsa (%5 ihtimalle) Churchill çıkar
- **Portakal Suyu:** Doğum günü bugüne denk geliyorsa direkt Portakal Suyu çıkar

### Enerji Haritası:
Algoritmanın ara puanlarını `cesaret`, `dinginlik`, `nese`, `ferahlık` olarak normalize et ve UI'da göster:
```
❤️ Cesaret: %82
🌿 Dinginlik: %45
☀️ Neşe: %71
🌊 Ferahlık: %63
```
Bu puanlar algoritmadan türetilir — tamamen eğlence amaçlı.

## Adım 3: MoodFinderSection Yeniden Yazımı
**Dosya:** `src/components/MoodFinderSection.tsx` (YENİDEN YAZ)

### Akış (6 Aşama):

**Aşama 1: Giriş (Mevcut form yapısı korunur)**
- Desktop: Tek form (Ad, Soyad, Doğum Tarihi)
- Mobile: 4 adımlı flow (welcome → name → birthday → ready)
- Tasarım: Mevcut glass portal stilinde, gradient underline inputlar

**Aşama 2: Loading — Sinematik Enerji Okuma (4-5 saniye)**
Canvas vortex yerine yazan metin animasyonu:
```
✓ Doğum enerjisi bulundu.     (0-1.2s)
✓ Karakter analiz ediliyor.     (1.2-2.4s)
✓ Lezzet eşleşmesi yapılıyor.   (2.4-3.6s)
✓ En yakın MOQ bulundu.        (3.6-4.5s)
```
Arka planda MOQ logosu nefes animasyonu + renk geçişi.
Metinler Framer Motion ile birer birer belirir (typewriter efekt).

**Aşama 3: Sonuç Ekranı**
- Başlık: "✨ MOQ Seni Seçti" (değil "Senin MOQ'un")
- İçecek ismi büyük font + emoji
- **Enerji haritası** (4 çubuk animasyonlu)
- Kişilik metni (hash ile seçilen varyasyon)
- İçecek detayları (malzemeler, dünya adı)
- Espri (içeceğe özel)
- Placeholder görsel alanı (boş kare, renk gradient + emoji, kullanıcı sonra ekler)

**Aşama 4: Nadir mesajlar (%5 ihtimalle)**
Sonucun altına rastgele:

- vb.

**Aşama 5: Mini espri havuzu**
