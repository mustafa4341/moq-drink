# Plan: Unify the site around the 8 real MOQ drinks + premium popup

## Goal
The 8 real MOQ drinks (Merida, Cool Lime, Redline, Sundrop, Sunset, Limonata, Churchill, Portakal Suyu) become the **single source of truth** for the entire site (Carousel, mobile popup, desktop ProductWorlds, Mood Finder). The 4 placeholder drinks (Blue Mojito, Berry Boost, Passion Breeze, Lime Fresh) and the `product-data.ts` file are removed entirely. A premium mobile popup is built per your specification. Desktop keeps its cinematic "scroll into the drink's world" behavior, expanded to 8 worlds, with per-drink anchors so clicking a card transports you to that specific world.

## Architecture decision: one schema
Extend the existing `MoqDrink` interface in `src/lib/drinks.ts` with the UI fields the Carousel/popup/ProductWorlds need, populated from your pasted spec. Keep all existing fields (`personalities`, `joke`, `resultTheme.*`, `traits`, `worldName`, etc.) untouched so the Mood Finder and mood algorithm keep working with zero changes.

## File-by-file changes

### 1. `src/lib/drinks.ts` — expand the schema + fill content (core change)
Add these fields to `MoqDrink`:
- `displayWorld: string` — cinematic world name ("YEŞİL VADİ", "ALTIN TROPİKLER", …) — the "Dünya Adı"
- `tagline: string` — 2-3 word character subtitle ("Enerji Dolu", "Sade ve Doğal", …)
- `story: string` — the 2-3 line narrative
- `moodLabels: string[]` — mood words (["Enerjik","Pozitif","Cesur"])
- `flavorNotes: string[]` — taste notes (["Portakal","Mango","Limon","Soda"])
- `moqMessage: string` — the single MOQ sentence ("Plan yapmayı bırak...")
- `accentColor: string` — hex (already effectively exists as `color`, but explicit)
- `textColorClass: string` — Tailwind class (`text-brand-green-text`, etc.)
- `bgStyle: string` — glass rgba string for cards
- `bgGrad: string` — Tailwind gradient classes for world backgrounds (e.g. `from-amber-100/60 via-orange-50/30 to-amber-200/40`)
- `atmosphere: { colorPrimary; colorSecondary; particleType; particleCount; particleIntensity }` — for the popup + worlds (reused from the old `Drink`/`Atmosphere` type, moved here)
- `bgDescription: string` — short description of the background mood ("Açık yeşil gradient, sis, yapraklar") used by the world ambient layer

Fill all 8 drinks from your pasted spec (world, color, tagline, story, mood, taste, MOQ message). **Keep `worldName`, `personalities`, `joke`, `resultTheme`, `traits` exactly as they are** — Mood Finder and mood-algorithm.ts depend on them.

Note: image paths stay as-is (`/images/merida.png`, etc.). Since those files don't exist yet, the components render a premium gradient+emoji placeholder when the image is missing (see #3/#4), and you can drop in real photos later by only changing the path here.

### 2. `src/lib/product-data.ts` — DELETE
Delete the file and its `Drink` / `Atmosphere` / `PRODUCTS` exports. Update all 3 importers:
- `src/app/page.tsx` — import `MOQ_DRINKS, type MoqDrink` from `@/lib/drinks` instead; `handleDrinkClick` searches `MOQ_DRINKS`.
- `src/components/InfiniteCarousel.tsx` — import from `@/lib/drinks`.
- `src/components/BottomSheet.tsx` — `Drink` → `MoqDrink`.

Rebuild the content to your exact layout:

Premium staggered animation (Framer Motion variants, ~respects `prefers-reduced-motion`):
- Background cross-fades to `drink.color` over 300-400ms via a tinted overlay
- Giant `displayWorld` typography behind content at ~5% opacity

- Mood labels appear with a soft glow
- MOQ message appears last
- CTA ends with a gentle "breathing" pulse (reuse `moq-breathe` keyframe already in globals.css)

Image handling: render the real `<Image>` when the file exists; otherwise a premium placeholder = the drink's own `gradient` + large `emoji` floating + particles (SheetParticles already exists). Keep ALL existing accessibility: focus trap, ESC, swipe-down dismiss, backdrop click, body lock, ARIA dialog.

### 4. `src/components/InfiniteCarousel.tsx` — 8 drinks, dual layout
- Source: `MOQ_DRINKS` (8). Desktop GSAP infinite-loop track now wraps 8 cards (×4 duplication for seamless loop). Mobile changes from a 2×2 grid → **horizontal scrollable row of 8 cards** (`overflow-x-auto scrollbar-none`, snap) per your request ("tek sıra halinde mobilde ... yana kaydırma ekleriz 8 taneye tamamlayalım").
- `ProductCard` (desktop) click handler: instead of scrolling to `#worlds-section` generically, smooth-scroll to the **specific world** `#world-<id>` (e.g. `#world-merida`). New prop `onCardClick(id)`.

### 5. `src/components/ProductWorlds.tsx` — expand 4 → 8 worlds (desktop-only, unchanged)
- Replace the local hardcoded `worldsData` (4) with `MOQ_DRINKS` (8). Map fields: `world.name`→`drink.name`, `themeTitle`→`drink.displayWorld`, `tagline`→`drink.tagline`, `description`→`drink.story`, `image`→`drink.image`, `bgGrad`→`drink.bgGrad`, `textColor`→`drink.textColorClass`, `particleColor`→`drink.resultTheme.particleColor`.
- Add a per-world anchor id `world-<drink.id>` on each `WorldSection` so desktop carousel clicks can target it. The section keeps its `id="worlds-section"` wrapper.
- Generalize the 4 hardcoded ambient-effect blocks (`world.id === 1/2/3/4`) into a single parameterized ambient layer driven by `drink.color`/`drink.bgDescription` so all 8 worlds feel distinct without 8 bespoke copies.
- Image placeholder: when the file is missing, show gradient + emoji + particles instead of a broken image (same treatment as the popup).
